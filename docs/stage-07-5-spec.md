# CPCV Stage 7.5 辞書フィルター 正式仕様

## 1. 目的

全コメントを外部AIへ送らずに。不適切表現の大半を組織管理の辞書で同期判定します。

辞書だけで確定できない表記ゆれは承認待ちへ送ります。授業設定が許可する場合だけAI判定を追加します。

## 2. 三層構造

1. 入力検証
   - 文字数
   - URL
   - 制御文字
2. 辞書フィルター
   - 正規一致
   - 記号挿入除去後の一致
   - 片仮名と平仮名の統一
   - 限定的な見た目類似文字
   - 編集距離による曖昧一致
3. AI補助
   - `off`: 使用しない
   - `ambiguous`: 曖昧一致だけ
   - `all`: 全投稿

## 3. 用語データ

用語ごとに次を保存します。

- 検閲用語
- 種類
- 検閲レベル 1から5
- match mode
- 表記ゆれ検出の有無
- 有効状態

種類:

- 下ネタ・性的表現
- 暴言・下品な表現
- 侮辱・嫌がらせ
- 差別的表現
- 暴力・脅迫
- 政治的発言
- 個人情報
- 宣伝・スパム
- 違法行為
- 独自分類

## 4. レベルと処理の分離

用語側は危険度だけを持ちます。

組織policy側が種類ごとに次の閾値を決めます。

- 承認待ち
- 伏字
- 投稿拒否

順序制約:

```text
承認待ち ≤ 伏字 ≤ 投稿拒否
```

空欄はその処理を使用しません。

政治的発言は既定で無効です。伏字と投稿拒否の既定閾値もありません。

## 5. 判定

### 確定一致

`strict`または`compact`一致です。

policyに従って承認待ち。伏字。投稿拒否を同期実行します。

### 曖昧一致

`confusable`または`fuzzy`一致です。

誤検出防止のため。直接伏字や投稿拒否にはしません。

必ず`review`として保存します。AI経路が`ambiguous`または`all`ならAI jobを作ります。

## 6. 回避表記

次を処理します。

- Unicode NFKC
- 英字の小文字化
- 片仮名から平仮名
- zero-width。format。combining markの除去
- 記号と空白を除いたcompact表現
- 一部のLatin。Cyrillic。Greek。leet類似文字
- 長さ4以上の語に対する限定的な編集距離

例:

```text
ちんこ
ち、んこ
チンコ
ち ん こ
```

同じ辞書語として検出できます。

編集距離一致は最大64候補です。語長に応じて距離1または2だけを許可します。

## 7. 原文と表示文

原文は`comments.message`へ保存します。

伏字後の表示文は`comments.display_message`へ保存します。

投影画面とRealtimeには表示文だけを送ります。教員画面は原文。表示文。判定根拠を確認できます。

## 8. AIとの関係

辞書フィルターはAIを置き換えるものではありません。

- 確定一致はAIを使わない
- 曖昧一致だけAIへ送れる
- AI結果は参考情報
- AIは人間のmoderation stateを自動変更しない
- 個人情報guardは従来どおりAI送信前に動く

## 9. 既定値

- 組織policyは全種類無効
- 授業辞書フィルターは無効
- AI経路は`ambiguous`
- 伏字文字は`＊`
- 用語上限は組織ごとに2000語
- fuzzy候補上限は投稿ごとに64語

## 10. 変更禁止範囲

- Student account追加
- 既存コメント原文の破壊
- AIによる自動非表示
- WebSocket sequence契約変更
- PDFのserver保存
- 既存migrationの書換え
- Stage 8機能の先行実装
