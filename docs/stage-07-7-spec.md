# CPCV Stage 7.7 日英検閲パック・翻訳前後検閲 正式仕様

## 1. 目的

Stage 7.6の汎用多言語辞書を運用可能な範囲へ絞る。

第一対応言語は日本語と英語だけとする。

- 日本語と英語は組織辞書で同期判定する。
- 日英以外または言語判定不能の文章は安全側で承認待ちにする。
- `ai_review`設定時だけ日英以外をAIの参考判定へ送る。
- AIは手動moderation stateを変更しない。
- 翻訳が有効な場合は翻訳前の原文と翻訳後の表示文を別々に検閲する。
- 翻訳結果だけを理由に原文コメントを削除または非表示にしない。

## 2. 処理順序

```text
学生投稿
  ↓
入力検証
  ↓
日本語・英語の辞書照合と言語判定
  ├─ reject → 保存しない
  ├─ mask → 原文と伏字文を分離して保存
  ├─ review → pendingで保存
  └─ allow → visibleまたは既存事前承認設定に従う
  ↓
日英以外の場合
  ├─ ai_review → pending + AI参考判定job
  ├─ review_only → pendingのみ
  └─ allow → 組織が明示した場合だけ通過
  ↓
人間が表示可能と判断
  ↓
翻訳設定と翻訳要否を確認
  ↓
Workers AI翻訳
  ↓
翻訳先言語の日英辞書を再適用
  ├─ allow → 翻訳を表示
  ├─ mask → 伏字化した翻訳だけ表示
  ├─ review → 翻訳を保留
  └─ reject → 翻訳を保留
```

## 3. 日英以外の扱い

日英以外を自動拒否しない。

既定値は`ai_review`。

```text
日英以外
→ 原文をpendingで保存
→ AI moderationを参考として実行
→ 教員が承認・非表示・削除を判断
```

通常の授業AI判定が無効でも`ai_review`は日英以外のコメントにmoderation jobを作る。ただし組織AI全体が無効なら外部AIを呼べないためpendingのまま人間確認へ送る。

## 4. 翻訳要否

次を全て満たす場合だけ翻訳jobを作る。

- 組織AIが有効
- 授業翻訳が有効
- コメントが表示可能
- 原文言語がneutralではない
- 原文言語と翻訳先言語が異なる
- 翻訳先が`ja`または`en`
- PII guardで停止されていない
- 日次上限内

pendingコメントは翻訳しない。承認または復元後に改めてjobを作る。

## 5. 翻訳後検閲

翻訳先が日本語なら日本語辞書と`und`用語を使用する。

翻訳先が英語なら英語辞書と`und`用語を使用する。

保存を分離する。

- `translations.translated_text`: AIが返した原翻訳
- `translations.display_text`: 表示可能な翻訳または伏字後翻訳
- `translations.filter_action`: `allow` `mask` `review` `reject`

ViewerとRealtimeには`allow`または`mask`の`display_text`だけを送る。`review`と`reject`は管理画面に保留として表示する。

## 6. 辞書パック

- `ja-core-v1`: 39語
- `en-core-v1`: 50語
- 政治的発言は収録しない
- 組織へコピーして利用する
- コピー後は管理者が変更・無効化・削除できる
- 導入は冪等
- 組織辞書の上限2,000語を超える場合は導入を拒否する

辞書パックは完全な言語資源ではない。教育現場向けの初期値であり誤検出と見逃しの測定が必要。

## 7. 言語判定

確定的な辞書一致がある場合は辞書語の言語を優先する。

それ以外はUnicode Scriptと保守的な英語語彙判定を使用する。

- 仮名を含む → 日本語
- 日本語として登録した漢字短語 → 日本語
- 明確な英語機能語を含むASCII Latin文 → 英語
- 非ASCII Latin。韓国語。中国語。アラビア語。その他script → 日英以外
- 漢字だけで日本語と中国語を判別できない → 日英以外
- `no puta`など共通短語を含む曖昧Latin文 → 日英以外

短文の自動言語判定は完全ではない。誤って英語と断定するより日英以外として承認待ちへ送る。

## 8. 変更しない境界

- Student匿名参加
- コメント本文の正本
- 手動moderation transition
- WebSocket sequenceとcatch-up
- AI判定の参考情報という位置付け
- PDFをserverへ保存しない方針
- 組織固定session
- メール認証
