# Stage 7.8 デバッグ報告

## 発見と修正

### 1. 語数不足

日本語39語。英語50語では派生語。脅迫句。複合罵倒を十分に扱えなかった。合計500語へ拡充した。

### 2. 一律導入による誤検出

解剖学。性教育。報道。引用で使われる語を基本パックへ混ぜると誤検出が増える。文脈注意パックへ分離した。

### 3. 旧pack更新不能

従来の`INSERT OR IGNORE`では導入済みpackのseverityや境界設定を更新できない。source keyを使ったUPSERTへ変更した。

### 4. 手動編集の上書き

管理者が変更した語はpackとの紐付けを解除する既存仕様を維持した。更新対象から除外した。

### 5. 削除語の復活

再導入で利用者が削除した語を復活させないようにした。

### 6. source key重複

組織。pack。term keyの部分UNIQUE INDEXを追加した。

### 7. 静的mirror不一致

管理画面を更新した後に`public/_admin_spa.html`が旧状態のままだった。精密境界検査で検出して同期した。

### 8. Stage 7.7境界検査の旧前提

後続migration 0014を誤って拒否する検査と`INSERT OR IGNORE`固定検査をappend-only互換へ修正した。
