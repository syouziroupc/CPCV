# CPCV Stage 7.8 辞書パック拡充 正式仕様

## 1. 目的

Stage 7.7の日本語39語。英語50語は動作確認用の初期値としては使える。しかし実授業での見逃しを抑えるには不足する。

Stage 7.8では語数だけを増やさない。誤検出と文脈の違いを考慮して次の二層へ分ける。

| 層 | 用途 | 既定導入 |
|---|---|---|
| 基本パック | 強い罵倒。露骨な性的俗語。脅迫。差別的蔑称 | 推奨と厳格 |
| 文脈注意パック | 教育。報道。引用では正当な場合もある語 | 厳格だけ |

政治的発言は両層へ含めない。

## 2. 収録数

| パック | version | 語数 |
|---|---:|---:|
| 日本語 基本 | 2 | 128 |
| 日本語 文脈注意 | 1 | 101 |
| 英語 基本 | 2 | 161 |
| 英語 文脈注意 | 1 | 110 |
| 合計 | ― | 500 |

推奨設定は基本パック289語を使用する。厳格設定は文脈注意211語も追加する。

## 3. 処理方針

- 基本パックは種類別閾値により伏字または投稿拒否まで到達できる。
- 文脈注意パックはseverity 1または2だけとする。
- 文脈注意パックだけで自動投稿拒否に到達しない。
- 厳格設定でも文脈注意語は原則として承認待ちにする。
- 人間のmoderationを最終判断とする。
- AIは曖昧一致または日英以外の参考判定だけに使う。

## 4. 日本語照合

- NFKC
- 片仮名から平仮名
- 空白。句読点。ゼロ幅文字の挿入除去
- 短い衝突しやすい語は単語境界を使用
- 確定できない一文字程度の変形は承認待ちとAI参考判定

## 5. 英語照合

- 全用語に単語境界を使用
- `class`内の短い語などを部分一致させない
- 空白や記号を挿入した回避表記は正規化して検出
- fuzzy一致は直接拒否せず承認待ちにする

## 6. パック更新

- Stage 7.7の日本語39キーと英語50キーを保持する。
- 未編集のpack管理語はversion 2へ更新する。
- 管理者が編集した語は`source_pack`が解除されるため上書きしない。
- 管理者が削除したpack語は再導入しても復活させない。
- 再導入は冪等とする。

## 7. 用語選定

- 放送基準を安全方針の参考にする。
- 公開されている日英の不適切語リストを候補集合として参照する。
- 候補をそのまま一括投入しない。
- 政治語。一般語との衝突が大きい短語。地域差が大きい語は除外または文脈注意へ移す。
- 脅迫文は単語だけでなく句として基本パックへ入れる。
- 定期的に誤検出。見逃し。利用頻度を監査する。

## 8. 変更禁止範囲

- コメント原文
- 手動moderation state
- Realtime sequence schema
- AIが自動非表示にしない契約
- 翻訳前後検閲
- 日英以外を承認待ちへ送る契約
