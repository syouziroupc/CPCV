# CPCV 0.8.10 セキュリティ・UI再監査

## 判定

**条件付き公開可能**です。今回確認した不具合と直接的な公開リスクは修正しました。ただし本番公開前にCloudflare上で確認すべき項目が残っています。

## 今回修正したセキュリティ項目

- QR生成APIを同一originの学生参加URL `/j/:publicCode` だけに制限
- 表示名でもURL。domain。IP形式を拒否
- コメントと表示名のUnicode双方向制御文字を拒否
- 学生の投稿制限を共有IPではなく参加者token単位へ変更
- rate limit keyをHMAC化しraw tokenを残さない
- 参加者Cookieの有効期間を30日から8時間へ短縮
- OwnerとAdminから手動maintenance APIを実行できないよう削除
- アカウント削除時にメール。ログインID。表示名。password情報を匿名化
- 全responseへ`Referrer-Policy: no-referrer`を追加
- QR SVGへsandbox CSPを設定
- 動的`innerHTML`を廃止
- 投影画面を新しいtabで開く際に`noopener,noreferrer`を使用
- 端末内コメントの上限を10,000件から2,000件へ削減
- secret。認証情報。private keyの混入検査を実施

## 本番公開前に残る項目

### 高

1. **password hash**
   - 現在はPBKDF2-HMAC-SHA256 100,000回です。
   - local測定は100,000回約20ms。600,000回約123msでした。
   - WorkerのCPU枠を確認せず600,000回へ変更するとloginが失敗する可能性があります。
   - staging Workerで実測してschemeを更新する必要があります。

2. **MFA未実装**
   - OwnerとAdminにMFAがありません。
   - 公開後に組織管理機能を広く提供する前にPasskeyかTOTPが必要です。

3. **公開自己登録と利用量上限**
   - 誰でもOwner組織を作成できます。
   - 授業数。コメント数。AI処理数の組織quotaが不足しています。
   - 一般公開時は招待制かquota実装が必要です。

4. **実Cloudflare設定未確認**
   - repositoryの`wrangler.toml`にはDB_V2の実IDとRate Limiting binding IDがありません。
   - 実値を入れたstaging設定でdry-run。migration。受入試験が必要です。

### 中

- 公開授業コード照会GETに専用rate limitがない
- `EMAIL_AUTH_REQUIRED=0`でlegacy login IDを許可している
- 投影画面の端末ログ保存を完全に無効化する共用端末modeがない
- AI送信前の個人情報検出はregex中心で完全ではない
- CSPの`style-src`に`unsafe-inline`が残る
