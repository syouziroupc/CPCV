# CPCVのバージョン命名規則

CPCVは三つの整数をピリオドで区切る独自方式を使用します。Semantic Versioningの互換性規則は適用しません。

```text
開発系列.大きな更新回.小さな更新回
```

現在の版は`0.8.10`です。

- 小規模な修正と改善は`0.8.11`、`0.8.12`のように末尾を増やします。
- 機能構成を大きく変更する版は`0.9.1`へ移ります。
- さらに次の大きな系列は`0.10.1`へ移ります。
- 各区画は整数です。`0.10.1`は`0.9.99`より新しい版として扱います。
- 末尾が9に達しても繰り上げません。`0.8.9`の次に`0.8.10`を使用できます。
- Git tagは`v0.8.10`の形式にします。

## 更新例

```text
0.8.10  現在版
0.8.11  軽微な修正
0.8.12  追加修正
0.9.1   次の大規模更新
0.9.2   その修正版
0.10.1  次系列の大規模更新
```

比較処理では文字列比較を使いません。各区画を整数として左から比較します。
